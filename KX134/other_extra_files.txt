imu_conversion is the physics layer.

It must:

1️⃣ Take raw sensor counts (int16_t)
2️⃣ Know what range is configured (8g / 16g / 32g / 64g)
3️⃣ Apply correct scale factor
4️⃣ Apply calibration offsets (optional but recommended)
5️⃣ Output usable physical units

in i2cbus.cpp
int written = i2c_write_timeout_us(_instance,
                                       device_addr,
                                       buffer,
                                       2,
                                       false,
                                       _timeout);
Pico SDK function. Parameters:

_instance — which I2C hardware block (i2c0 or i2c1)
device_addr — the 7-bit I2C address of the target sensor
buffer — pointer to the data to send
2 — number of bytes to send
false — nostop flag. false means send a STOP condition after the write, ending the transaction normally. true would hold the bus for a follow-up read (not needed here)
_timeout — microseconds before giving up
